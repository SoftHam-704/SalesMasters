# SalesMasters System Manual
> Internal Knowledge Base for AI Assistant & User Training
> Version 1.0 - Created for "Professor Mode"

## 1. Visão Geral do Sistema (Philosophy)
O SalesMasters não é apenas um CRM; é um **Sisteam de Inteligência Comercial**.
O objetivo não é apenas "cadastrar pedidos", mas **maximizar o Sell-Out** e a **Positivação** da carteira.
Todos os módulos giram em torno da **eficiência do vendedor**.

**Regra de Ouro:** "Se não ajuda a vender, não deveria estar na tela."

---

## 2. Estrutura de Navegação (Menu)

### 2.1 Dashboard (Visão do Piloto)
- **Função:** Mostrar o status atual da meta e onde estão os incêndios.
- **KPIs Críticos:**
  - *Meta vs Realizado:* Onde estou hoje.
  - *Projeção:* Onde estarei se continuar assim.
  - *Curva ABC:* Quais clientes grandes pararam de comprar?

### 2.2 Pedidos de Venda
- **Fluxo de Trabalho:**
  1. Selecionar Cliente (Busca inteligente por Nome/Fantasia).
  2. Selecionar Tabela de Preço (O sistema sugere a tabela padrão do cliente).
  3. Inserir Itens (Verificar Estoque em tempo real).
  4. Finalizar: O pedido cai no ERP para faturamento.
- **Dica Ninja:** Use o atalho `F2` para carregar o "Magic Order" (Pedido baseado no histórico do cliente).

### 2.3 CRM (Kanban & Pipeline)
- **Filosofia:** "Zero Fricção". O vendedor não deve preencher formulários.
- **Colunas do Funil:**
  1. **Novo:** Cliente importado ou Lead frio.
  2. **Contato:** Tentativa de ligação/zap feita.
  3. **Negociação:** Proposta enviada.
  4. **Fechamento:** Aguardando assinatura/pagamento.
- **Ações Rápidas:**
  - Clicar no ícone do WhatsApp no card abre a conversa e já registra o log automaticamente.

### 2.4 Sell-Out (A Venda Real)
- Monitora o que o cliente (distribuidor) vendeu para o consumidor final.
- Se o Sell-Out está baixo, não adianta "empurrar" mais produto (Sell-In).

## 3. Inteligência Artificial e Automação
### 3.1 Assistente IA
- Localizado no menu Utilitários.
- **Capacidade:** Responder perguntas sobre clientes, produtos e metas.
- **Treinamento:** Lê este manual e a base de dados em tempo real.

### 3.2 Magic Load (F2)
- Algoritmo que prevê o pedido ideal.
- Analisa: Giro médio + Estoque estimado + Sazonalidade.

---

## 4. Glossário Técnico
- **Positivação:** % da carteira que comprou no mês.
- **Churn:** Cliente que parou de comprar.
- **Ruptura:** Produto que faltou no cliente.
